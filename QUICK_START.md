# 快速启动指南

## 🎉 无需后端，开箱即用！

**本项目支持模拟数据模式，无需启动后端服务即可运行！**

- ✅ 登录无需验证
- ✅ 所有页面都有模拟数据
- ✅ 功能完全可演示
- ✅ 适合前端独立开发

## 5分钟快速上手

### 步骤 1: 安装依赖

```bash
cd /Users/vikesun/workspace/pages/merchant-client
npm install
```

### 步骤 2: 启动开发服务器

```bash
npm run dev
```

服务器将在 `http://localhost:3000` 启动。

### 步骤 3: 登录系统

打开浏览器访问 http://localhost:3000，使用以下测试账号登录：

#### 系统管理员账号
```
商户域名: test.yc365.com（或任意输入）
账号: system_admin（或任意输入）
角色: 系统管理员
密码: 123456（或任意输入）
```

#### 业务管理员账号
```
商户域名: test.yc365.com（或任意输入）
账号: business_admin（或任意输入）
角色: 业务管理员
密码: 123456（或任意输入）
```

> 💡 **重要提示**: 
> - 目前登录**不需要验证**，输入任意账号密码都可以登录
> - 系统会自动生成模拟的用户信息
> - 所有数据都是前端模拟数据，刷新后会重置
> - 查看 [MOCK_MODE.md](./MOCK_MODE.md) 了解更多

## 开发环境配置

### 模拟数据模式（默认）

**项目已默认启用模拟数据模式**，无需任何配置即可运行！

所有数据都是前端模拟的，包括：
- ✅ 用户登录
- ✅ 业务管理员列表
- ✅ 交易记录
- ✅ 用户数据
- ✅ API Keys
- ✅ 积分信息
- ✅ 所有其他数据

### 切换到真实 API（可选）

如果要使用真实的后端 API：

1. 确保后端服务运行在 `http://localhost:8080`
2. 修改 `src/pages/Login/index.tsx` 恢复真实 API 调用
3. 参考 [API_INTEGRATION.md](./API_INTEGRATION.md) 对接接口

详细说明请查看 [MOCK_MODE.md](./MOCK_MODE.md)

## 常见问题

### Q: 安装依赖失败？

```bash
# 清除缓存重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### Q: 启动失败，端口被占用？

```bash
# 修改端口（在 vite.config.ts 中）
server: {
  port: 3001, // 改为其他端口
}
```

### Q: 登录失败？

**A**: 目前登录不需要后端验证，任意输入账号密码都能登录。如果还是失败，请检查浏览器控制台错误。

### Q: 页面显示空白？

1. 检查浏览器控制台错误
2. 确认 Node 版本 >= 18
3. 清除浏览器缓存
