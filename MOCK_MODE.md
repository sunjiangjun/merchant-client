# 模拟数据模式说明

## 概述

本项目支持**无后端运行模式**，可以在没有后端 API 的情况下独立运行，所有数据都使用前端模拟数据。

## 启用模拟模式

模拟模式**已默认启用**，无需额外配置。

### 特点

1. ✅ **登录无需验证** - 输入任意账号密码即可登录
2. ✅ **数据完全模拟** - 所有页面数据都使用模拟数据
3. ✅ **功能可演示** - 所有功能都可以正常操作
4. ✅ **网络错误静默** - 后端不可用时不会报错

## 模拟数据说明

### 登录

- 商户域名：任意输入
- 账号：任意输入
- 角色：选择系统管理员或业务管理员
- 密码：任意输入

登录后会自动生成模拟的用户信息和 Token。

### 建议测试账号

为了更好的演示效果，建议使用：

```
系统管理员：
  域名: test.yc365.com
  账号: system_admin
  密码: 123456

业务管理员：
  域名: test.yc365.com
  账号: business_admin
  密码: 123456
```

### 模拟数据内容

所有页面都会显示模拟数据：

- **业务管理员列表**: 3 个模拟管理员
- **交易记录**: 3 条模拟交易
- **Sign 配置**: 2 个模拟配置
- **商户资产**: 模拟资产数据
- **用户列表**: 50 个模拟用户
- **用户订单**: 20 条模拟订单
- **API Keys**: 3 个模拟 Key
- **积分信息**: 模拟积分数据
- **积分分配记录**: 30 条模拟记录

### 数据特点

- 所有 ID 都是随机生成
- 日期时间都是相对当前时间的随机值
- 数据在页面刷新后会重置
- 操作（创建、编辑、删除）会立即生效，但刷新后恢复

## 切换到真实 API

当后端 API 准备就绪后，只需：

1. 修改 `.env.development` 中的 API 地址

```env
VITE_API_BASE_URL=http://your-backend-api:port/api
```

2. 修改 `src/pages/Login/index.tsx`，使用真实的 API 调用

```typescript
// 恢复原来的登录逻辑
const { token, user } = await login(values);
```

3. 删除或注释掉模拟数据相关代码

## 开发建议

### 前端独立开发

使用模拟数据模式，前端可以：

- 独立开发和调试
- 演示功能和界面
- 进行 UI/UX 优化
- 编写单元测试

### 与后端联调

当需要与后端联调时：

1. 确保后端 API 服务运行在 `http://localhost:8080`
2. 后端接口遵循 `API_INTEGRATION.md` 中的规范
3. 逐步替换模拟数据为真实 API 调用

## 模拟数据文件

模拟数据定义在 `src/utils/mockData.ts` 中，您可以：

- 修改模拟数据的数量
- 调整数据的内容
- 添加新的模拟数据

## 常见问题

### Q: 为什么操作后数据没有保存？

A: 模拟模式下，所有操作都是前端模拟的，数据不会持久化。刷新页面后会恢复到初始状态。

### Q: 如何添加更多模拟数据？

A: 编辑 `src/utils/mockData.ts` 文件，增加或修改相应的模拟数据数组。

### Q: 登录时提示网络错误？

A: 这是正常的，因为没有后端服务。登录逻辑已经改为不依赖后端，可以直接登录。

### Q: 某些功能不能用？

A: 涉及到外部服务的功能（如钱包连接）需要额外的集成工作，当前只是界面展示。

## 优势

使用模拟数据模式的优势：

1. **快速启动** - 无需等待后端开发
2. **独立演示** - 可以向客户演示功能
3. **稳定开发** - 不受后端服务状态影响
4. **易于测试** - 可以控制各种数据场景
5. **低成本** - 无需部署后端服务

## 注意事项

⚠️ **生产环境务必禁用模拟模式**

在部署到生产环境前，请确保：

1. 配置正确的后端 API 地址
2. 恢复真实的 API 调用逻辑
3. 删除或禁用模拟数据代码
4. 进行完整的集成测试
